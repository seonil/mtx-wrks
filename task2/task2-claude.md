## 3. AI + 표준 템플릿 기반 개선 방안

### 📌 PHASE 1 — AI RFQ 자동 파싱 & 분류 (수신 단계)

| 항목 | 내용 |
|------|------|
| **적용 기술** | LLM 기반 문서 파싱 (OCR + NLP) |
| **핵심 기능** | 고객 RFQ PDF/Excel을 자동 분석하여 **제품 카테고리, 기술 요구사항, 수량, 납기, 품질 기준** 등을 구조화 추출 |
| **표준 템플릿** | `RFQ 접수 마스터 시트` — 고객명, 차종, 부품번호, 요구 스펙, 일정 등 30+ 필드 자동 매핑 |
| **기대효과** | 접수~분류 시간 **1.5일 → 0.3일** (반나절 내 완료) |

```
[고객 RFQ 메일/파일] → AI 파서 → 구조화된 요구사항 JSON
                                   → 자동 유사 프로젝트 매칭
                                   → 난이도·리스크 자동 등급화
```

### 📌 PHASE 2 — 통합 플랫폼 & 자동 Task 배분 (착수 단계)

| 항목 | 내용 |
|------|------|
| **적용 기술** | 워크플로 엔진 + Rule-based 자동 배분 |
| **핵심 기능** | RFQ 유형에 따라 **관련 부서에 표준화된 입력 템플릿을 자동 배포**, 마감일 역산하여 부서별 데드라인 자동 설정 |
| **표준 템플릿** | 부서별 전용 입력 폼 (아래 상세) |
| **기대효과** | 배분 시간 **2일 → 0.5일**, 누락 방지율 **99%+** |

#### 🗂 부서별 표준 입력 템플릿 체계

```
📁 RFQ Response Template Suite
│
├── 📋 [T-01] 영업 — 고객 요구사항 확인서
│   └── 고객 기본정보, SOP 일정, 수량 Forecast, 특수 조건
│
├── 📋 [T-02] 설계 — 기술 Feasibility 검토서
│   └── 기술 요구사항 대응표, Gap 분석, 개발 일정, 리스크 평가
│
├── 📋 [T-03] 품질 — 품질 요구사항 대응서
│   └── IATF/고객 규격 대비표, PPAP 계획, 공정능력 예측
│
├── 📋 [T-04] 원가 — 원가 산출 워크시트
│   └── 재료비, 가공비, 설비투자, 물류비, 마진 구조
│
├── 📋 [T-05] 공정 — 생산 Feasibility 검토서
│   └── 공정 흐름도(안), 설비 Capacity, 투자 소요
│
├── 📋 [T-06] 구매 — 핵심 자재 검토서
│   └── 주요 원자재 단가, 공급 리스크, 대체재 검토
│
└── 📋 [T-00] 통합 — RFQ Response Master (자동 조립)
    └── 위 T-01~T-06 데이터를 자동 통합·고객 포맷으로 변환
```

### 📌 PHASE 3 — 실시간 협업 대시보드 (진행 단계)

| 기능 | 설명 |
|------|------|
| **진행률 트래커** | 부서별 입력 완료율 실시간 시각화 (0% → 100%) |
| **자동 알림** | 마감 48h/24h 전 자동 Reminder, 미입력 시 팀장 Escalation |
| **크로스 참조** | 설계 변경 시 원가·품질 템플릿에 자동 연동 알림 |
| **이력 관리** | 모든 수정 이력 자동 기록, 버전 충돌 원천 방지 |

### 📌 PHASE 4 — AI 원가 추정 & 검증 (핵심 단계)

| 항목 | 내용 |
|------|------|
| **유사 견적 매칭** | 과거 수주 DB에서 유사 부품·스펙 자동 검색, 원가 구조 참조값 제시 |
| **AI 원가 모델** | 부품 특성(크기, 소재, 공정 수, 품질 등급)을 입력하면 **원가 범위를 자동 추정** |
| **이상치 탐지** | 입력된 원가가 과거 데이터 대비 ±20% 이상 벗어나면 자동 경고 |
| **시나리오 분석** | 수량·환율·원자재가 변동에 따른 원가 시뮬레이션 자동 제공 |

```
[부품 스펙] + [과거 수주 DB 300건+] → ML 모델 → 추정 원가 범위
                                               → 경쟁력 분석 리포트
                                               → 이상치 Alert
```

### 📌 PHASE 5 — 자동 문서 생성 & AI 검증 (완료 단계)

| 항목 | 내용 |
|------|------|
| **자동 문서 조립** | T-01~T-06 입력 데이터를 고객 지정 포맷(현대·기아, VW, GM 등)으로 자동 변환 |
| **AI 정합성 검증** | 설계 스펙 ↔ 원가 ↔ 공정 간 데이터 불일치 자동 탐지 |
| **체크리스트 자동화** | 제출 전 필수 항목 누락 여부 자동 점검 (고객별 필수 요구사항 DB 연동) |

## 4. 도입 로드맵

| 단계 | 기간 | 주요 활동 | Quick Win |
|------|------|----------|-----------|
| **Stage 1** | 1~2개월 | 표준 템플릿 체계 수립 + 통합 플랫폼(SharePoint/Notion) 구축 | 버전 혼란 해소, 취합 시간 30%↓ |
| **Stage 2** | 3~4개월 | 워크플로 자동화 + 대시보드 구축 | 부서간 사일로 해소, 누락 방지 |
| **Stage 3** | 5~8개월 | AI RFQ 파싱 + 원가 추정 모델 개발 | 리드타임 50%↓ 달성 |
| **Stage 4** | 9~12개월 | AI 문서 자동생성 + 지속 학습 체계 | 리드타임 63%↓, Win-rate 향상 |

## 💡 실행 시 핵심 성공 요인

1. **경영진 스폰서십** — RFQ 프로세스 혁신을 전사 과제로 선포하고, 부서 KPI에 협업 지표 반영
2. **데이터 축적이 먼저** — AI 모델 성능은 과거 견적 데이터 품질에 비례, Stage 1부터 체계적 데이터 수집 시작
3. **고객사별 커스터마이징** — 현대·기아, VW, GM 등 고객별 제출 포맷이 다르므로, 템플릿 변환 룰을 고객별로 사전 정의
4. **점진적 도입** — 한 번에 전체 전환보다 **특정 고객/제품군에서 파일럿** 후 확산하는 전략이 리스크 최소화
5. **현업 참여 설계** — 각 부서 실무 담당자가 템플릿 설계에 참여해야 실제 활용도 보장

> **핵심 메시지**: 표준 템플릿으로 **"무엇을 채울지"를 명확히** 하고, AI로 **"빠르고 정확하게 채우는 것"을 자동화**하면, 전장부품회사의 RFQ 대응은 속도와 품질 두 마리 토끼를 모두 잡을 수 있습니다.
